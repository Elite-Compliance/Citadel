# Citadel Source Automations

This unattended runner exports the eleven protected Receivables saved views plus the Deposits report from Blaze, validates them locally, publishes them to protected Google Drive/Sheets locations, and runs the existing Citadel importers.

## Safety behavior

- Blaze UUID from Job Link remains the primary key.
- All eleven Receivables reports must be present before publication.
- Specialized statuses must also exist in Receivables Aging.
- Duplicate UUIDs and malformed Job Links stop the run.
- Empty specialized reports remain valid.
- Exact Deposit Report duplicates are retained for review by the payment importer.
- New files are staged before the prior Drive report set is archived.
- A failed run leaves the last successful Citadel data available.
- Exported business data exists only on the temporary GitHub runner and protected Google files. It is never committed or uploaded as a workflow artifact.

## Schedule

The workflow runs at 7:00 AM, 12:00 PM, and 3:00 PM America/New_York, Monday through Friday. Its paired UTC schedules cover daylight-saving changes, and the runner exits without accessing Blaze unless the local Eastern hour matches one of those three times.

Scheduled pulls remain safely disabled until the repository variable `LIEN_AUTOMATION_ENABLED` is set to `true`. Manual **Run workflow** tests work before that switch is enabled.

## Required GitHub Actions secrets

- `BLAZE_EMAIL`: dedicated Blaze automation-user email.
- `BLAZE_PASSWORD`: password for that dedicated user.
- `BLAZE_TOTP_SECRET`: optional authenticator seed when Blaze requires a rotating verification code.
- `GOOGLE_SERVICE_ACCOUNT_JSON`: Google service-account key JSON. Share the protected Liens Drive folder and the Payment spreadsheet with the service-account email as Editor.
- `CITADEL_AUTOMATION_TOKEN`: generated by running `setupLienAutomation` once in the Citadel Apps Script project.

Do not commit credentials, cookies, downloaded reports, or Google keys to the repository.

## Removify Reviews

The Reviews runner signs in to Removify, exports the complete Monitoring CSV, validates all nine source columns, and publishes the verified snapshot to the protected `ReviewRecords` tab. Existing Review notes, alerts, and follow-ups are never replaced. Import and automation logs retain the row totals and any duplicate findings.

The automated schedule is 7:30 AM America/New_York, Monday through Friday. Paired UTC schedules cover daylight-saving changes; the runner verifies Eastern time before accessing Removify. Scheduled runs remain disabled until the repository variable `REVIEWS_AUTOMATION_ENABLED` is set to `true`. Manual **Run workflow** tests are available first.

Required GitHub Actions secrets:

- `REMOVIFY_EMAIL`: the Removify automation-user email.
- `REMOVIFY_PASSWORD`: the Removify automation-user password.
- `GOOGLE_SERVICE_ACCOUNT_JSON`: the same protected Google service-account JSON used by the other Citadel imports. Share the Reviews spreadsheet with its service-account email as Editor.

The import rejects missing columns, invalid ratings, blank required identity fields, empty exports, and suspiciously small snapshots. Every valid source row is retained; duplicate totals are recorded in the protected import logs.
